## 📝 使用建议

### 1. 需求评审会议

在需求评审时，打开 `raw_context` 查看原始对话：

```python
import json

with open('output/requirements_with_context.json') as f:
    data = json.load(f)

# 查看特定需求的原始上下文
req = data['requirements'][0]
print(req['raw_context']['conversation_context'])
```

### 2. 开发人员查阅

开发人员可以在任务清单中查看原始需求描述：

```markdown
## TASK-0004-01: 需求理解与方案设计

**来源需求**: REQ-0004 - 食物库推荐系统

**原始需求描述**: 
食物库推荐系统-  

**飞书原始字段**:
- 优先级: P1
- 状态: 待启动
- 负责人: 马健文

**验收标准**:
- [ ] 方案设计完成
```

### 3. 追溯原始对话

如果需求来源于微信聊天记录，可以通过 `raw_context` 追溯到具体的对话时间和内容。
