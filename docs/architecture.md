## 模块详细设计

### 1. 收集器模块 (Collectors)

#### 职责
- 连接群聊数据源
- 按时间范围收集消息
- 将原始数据转换为统一的消息模型

#### 关键类

**BaseCollector**
```python
class BaseCollector(ABC):
    async def connect(self) -> bool
    async def collect_messages(group_id, start, end) -> List[ChatMessage]
    async def get_groups() -> List[dict]
```

**WeChatCollector**
- 读取微信解密后的 SQLite 数据库
- 支持多数据库文件（message_0.db, message_1.db...）
- 消息表通过 MD5 hash 命名

**QQCollector**
- 支持 QQ NT 版本数据库
- 支持旧版 QQ 数据库格式

### 2. 分析器模块 (Analyzer)

#### 职责
- 从消息中提取反馈
- 对反馈进行分类
- 合并相似反馈

#### 处理流程

```
Messages
    ↓
[FeedbackExtractor] → LLM Prompt → JSON → Feedback Objects
    ↓
[FeedbackClassifier] → Keyword Rules → Tagged Feedbacks
    ↓
[FeedbackDeduplicator] → Similarity Check → Merged Feedbacks
    ↓
AnalysisResult
```

#### LLM 提示词设计

系统提示词：
```
你是一个专业的软件反馈分析助手。你的任务是从用户群聊消息中
提取与软件「{software_name}」相关的需求和 Bug 报告。

提取规则：
1. 只提取与 {software_name} 相关的反馈，忽略无关闲聊
2. 区分 Bug（功能异常）和 Feature（新需求）
3. 合并同一用户连续发送的关于同一问题的多条消息
4. 提取关键信息：问题描述、复现步骤、期望结果
5. 评估优先级：critical/high/medium/low
6. 提取关键词和标签

输出格式必须是 JSON...
```

### 3. 导出器模块 (Exporters)

#### 职责
- 将分析结果导出到目标平台
- 处理重复检测
- 格式化输出

#### 导出格式

**本地文件**
- Markdown: 人类可读的报告
- JSON: 完整数据，便于程序处理
- CSV: 表格形式，便于导入 Excel

**飞书多维表格**
- 字段映射到表格列
- 支持标签多选
- 支持状态流转

**Notion 数据库**
- 页面属性映射
- 支持标签系统
- 支持状态看板

## 数据流

### 完整流程

```
1. 用户输入
   └── 命令行参数 (platform, group, date, output)

2. 配置加载
   └── 环境变量 → 配置文件 → 默认配置

3. 收集阶段
   └── 连接数据库 → 获取群聊 → 查询消息 → 返回 ChatMessage[]

4. 分析阶段
   └── 提取反馈 → 分类 → 去重 → 返回 AnalysisResult

5. 导出阶段
   └── 检查重复 → 格式转换 → API 调用 → 完成

6. 结果展示
   └── 控制台输出 → 日志记录 → 报告文件
```

## 扩展性设计

### 添加新平台

1. 实现 `BaseCollector` 接口
2. 注册到 `collectors/__init__.py`
3. 在 CLI 中添加平台选项

### 添加新导出目标

1. 实现 `BaseExporter` 接口
2. 实现 `check_existing()` 方法
3. 在 CLI 中添加导出选项

### 自定义 AI 分析

1. 修改 `analyzer/extractor.py` 中的提示词
2. 或继承 `FeedbackExtractor` 重写提取逻辑

## 性能考虑

### 数据库查询
- 使用索引加速时间范围查询
- 批量读取避免内存溢出
- 连接池复用数据库连接

### AI 调用
- 批量处理消息，减少 API 调用次数
- 使用流式响应提高用户体验
- 添加超时和重试机制

### 去重算法
- Level 1: O(n) 哈希去重
- Level 2: O(n²) 相似度计算（小批量）
- Level 3: 异步比对历史数据

## 安全考虑

1. **数据隐私**: 用户聊天记录包含敏感信息
   - 本地处理，不上传原始消息到云端
   - AI 分析只发送必要内容

2. **API 密钥**: 安全存储 API 密钥
   - 配置文件不提交到 Git
   - 支持环境变量注入

3. **数据库访问**: 只读访问微信数据库
   - 不修改原始数据
   - 使用只读模式打开数据库
